DOCUMENTO TÉCNICO OFICIAL
Agente de Planejamento e Controle Financeiro Agrícola
Versão Técnica para Equipe e Codex
1. Arquitetura Geral do Sistema
* Backend: Node.js com TypeScript
* Infraestrutura: VPS Hetzner
* Banco principal: Supabase (Postgres)
* Integração WhatsApp: WAHA
* DNS e segurança: Cloudflare
* Gateway de pagamento: a definir (recomendado Pagar.me)
2. Estrutura do Projeto (Node + TypeScript)
* Separação por camadas: domain, application, infrastructure, modules
* Uso de arquitetura modular
* Separação clara entre regras de negócio e integrações externas
3. Domínio do Sistema
* Usuários vinculados a telefone e email
* Fazendas com múltiplas safras
* Planejamento por safra e cultura
* Categorias agrícolas configuráveis
* Transações financeiras (entrada/saída)
* Parcelamentos e cartões
* Assinaturas e planos
4. Regras de Negócio Críticas
* Parcelas impactam planejamento pelo valor total
* Fluxo de caixa impactado mês a mês
* Alertas em 50%, 80%, 100% e estouro
* Confirmação obrigatória para exclusão e edição
* Plano gratuito limitado a 5 lançamentos/dia


5. Banco de Dados (Postgres - Supabase)
* Valores armazenados em centavos
* Soft delete recomendado
* Auditoria obrigatória (create, update, delete)
* Timezone padrão: UTC com conversão para America/Sao_Paulo
6. API e Integrações
* Webhook WAHA para entrada de mensagens
* Endpoints REST para transações, planejamento e relatórios
* Webhook de pagamento para atualização de assinatura
7. Jobs e Automação
* Relatório diário às 7h (America/Sao_Paulo)
* Uso de node-cron
* Registro de histórico de relatórios enviados
8. Segurança e Confiabilidade
* Validação de input com Zod
* Logs estruturados com Pino
* Proteção contra mensagens duplicadas
* Auditoria completa de ações financeiras
9. Definição de Pronto (MVP)
* Cadastro via WhatsApp
* Criação de planejamento
* Registro de entradas e saídas
* Controle básico de cartão e parcelamento
* Relatório diário automático
* Sistema de assinatura funcional
